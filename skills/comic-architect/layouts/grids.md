# Grid Layouts

This engine defines the structural skeleton of the image. The layout determines how information is organized.

## 1. Comic & Narrative Grids
- **4-Panel Grid (2x2)**: Classic Yonkoma or comic strip. Equal weight to 4 steps. Good for "Setup -> Conflict -> Climax -> Resolution".
- **3-Panel Strip (Horizontal)**: Cinematic flow. Good for "Before -> Action -> Result".
- **9-Panel Grid (3x3)**: Dense information, "Brady Bunch" style, or micro-expressions.
- **Dynamic Action**: Angled panels, breaking borders, overlapping characters. High energy.

## 2. Information & Data Grids
- **Bento Grid**: Modular, rounded rectangles of varying sizes (1x1, 2x1, 2x2) fitted together like a bento box. Modern UI feel.
- **Timeline (Snake)**: S-shaped path winding down the page. Good for history or process steps.
- **Flowchart**: Node-based, connected by arrows. Hierarchical (Top-down) or Network (Mesh).
- **Comparison Split (50/50)**: Vertical or horizontal split. "Do vs Don't", "Old vs New".
- **Knolling (Flat Lay)**: Objects arranged at 90-degree angles, organized by type/color. No grid lines, but grid alignment.

## 3. Artistic Composition
- **Golden Spiral**: Focal point follows the Fibonacci spiral. Natural, pleasing flow.
- **Rule of Thirds**: Key elements placed at intersection points (1/3 lines).
- **Central Symmetrical**: Subject dead center, perfect symmetry. Wes Anderson vibe.
- **Negative Space Heavy**: Subject occupies small area (e.g., bottom right), vast empty space elsewhere. "Zen" or "Lonely" feel.
- **Isometric View**: 30-degree angle pseudo-3D. Good for maps, rooms, systems.

## 4. UI/App Layouts
- **App Icon Grid**: 5x2 or 4x3 array of icons. Uniform size and spacing.
- **Dashboard**: Header area + Sidebar + Main Content Area (Cards/Charts).
- **Mobile Screen**: Vertical 9:16 format, stacked elements (Header -> Hero Image -> List).
