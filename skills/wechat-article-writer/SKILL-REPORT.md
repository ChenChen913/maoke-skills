## 3. 核心工作流概览

```
用户输入
  |
  v
Phase 0: 两层智能判断 -----> 8 种入口路径（A-H，含文章对比）
  |                          **初始化/读取 creation-log.md**
  v（根据路径跳转）
Phase 1: 需求澄清 ---------> 5-8 个追问问题，一次一个，每个附示例
  |
  v
Phase 2: 市场调研 ---------> **日期确认** + **三层关键词搜索** + **读者视角调研** + **映射表输出**
  |
  v
Phase 3: 深度对话 ---------> 素材检索 + 风格预加载 + 大纲共创
  |                          **长文拆篇检测** (触发 references/series-split.md)
  v
Phase 4: AI 撰写初稿 ------> **模块化写作** + **叙述禁令** + 匹配风格 + 真实素材
  |
  v
Phase 5: 三轮审核 ---------> **主动触发** -> 内容审校(含时效性核查) -> 风格审校(6步) -> 细节打磨
  |                          **审核后主动询问微调或进入下一阶段**
  v
Phase 6: 首尾完善 ---------> 智能推荐文首文末模块
  |
  v
Phase 7: 媒体资源（配图 + 配视频） -> 图片双模式 + 视频双模式 + 汇总表
  |
  v
Phase 8: 标题与摘要 -------> 3-5 个标题候选 + 2-3 个摘要候选
  |
  v
Phase 9: 最终交付 ---------> Markdown 成稿 + 联动其他 Skill
```

**自动化特性**：
- 自动进度条
- 实时存档机制
- 上下文锚点与循环探测
- 长文自动拆篇提示

## 4. v1.8.0 新增核心特性

### 4.1 🔄 文章对比分析 (Article Compare)
- **触发**：用户提出"对比"需求或 Phase 0 识别。
- **流程**：收集文章 -> 四维对比（结构/观点/论据/风格） -> 综合推荐（直接用/融合）。
- **价值**：帮助用户从多篇 AI 稿件中取长补短，甚至融合出最优解。

### 4.2 ✂️ 长文拆篇规划 (Series Split)
- **触发**：大纲预估字数 > 3500 或模块 > 4 时主动提示。
- **流程**：确认拆分原则（逻辑拆/叙事拆） -> 生成分篇方案（标题/核心内容/钩子）。
- **执行**：单篇推进，自动添加系列导读，creation-log 记录系列进度。

### 4.3 🛠️ 流程健壮性修复（v1.7 继承）
- **强制状态追踪**：状态表追踪 Phase 状态。
- **禁止隐性交付**：Phase 4 后禁止直接交付。
- **信息时效性**：强制确认日期，搜索带年份。

## 5. 核心设计理念

### 5.1 设计来源

| 来源 | 核心贡献 |
|------|---------|
| 用户原始需求 | 5 步基础流程、追问式交互、三轮审核、配图标注 |
| 花叔方案 | 两层判断机制、Brief 概念、个人素材库（事前预防）、透明化思考、更系统的审校 |
| 宝玉方案 | 风格学习迭代、AI+脚本各司其职、模块化思维 |
| v1.2 用户反馈 | 用户优先级、步骤跳跃、分段修改、信息时效性、素材库管理、emoji 规则 |
| v1.3 用户反馈 | 叙述禁令、模块化写作、参考库扩展、模糊需求重置 |
| v1.4 用户反馈 | 自动化进度条、主动停顿询问、上下文锚点 |
| v1.5 用户反馈 | 循环探测、进度守恒、项目实时存档 (Creation Log) |
| v1.6 用户反馈 | 调研深度化、读者视角、调研-写作映射、交接备忘 |
| v1.7 用户反馈 | 流程健壮性修复（防跳跃）、信息时效性强制保障 |
| v1.8 用户反馈 | **[新]** 文章对比分析、长文拆篇规划 |

### 5.2 十一设计原则

1. **用户优先** -- 用户的自然语言指令永远优先于预设流程
2. **多追问、少填表** -- 所有问题附参考答案，降低用户门槛
3. **事前预防 > 事后补救** -- 用真实素材防止 AI 味，而非写完再改
4. **透明化思考** -- AI 每个决策都说明推理过程
5. **每步可跳过** -- 赶时间的用户不被流程绑架
6. **Token 优化** -- my-style.md 缓存 + 分段修改 + 按需读取
7. **渐进式披露** -- SKILL.md 精简，详细内容在 references/ 中按需加载
8. **模块化控制** -- 强制分段写作，杜绝长文失控与幻觉
9. **流程可视化** -- 实时进度条反馈，消除过程焦虑
10. **状态持久化** -- 实时文件存档，无惧超长对话遗忘
11. **调研落地化** -- 调研必须转化为写作指令，拒绝无效数据堆砌

## 7. 网络与异常处理

| 场景 | 处理方式 |
|------|---------|
| 搜索工具不可用 | 三级预案：切换工具 -> AI 知识库（标注截止日期） -> 诚实告知 |
| 会话中断 | 恢复协议：询问断点 -> **读取 creation-log.md** -> 继续 |
| 任何文章写作 | **模块化写作原则**：分模块撰写，每模块确认后再写下一模块 |
| 叙述性背景需求 | **叙述禁令**：检测到人味需求 -> 暂停 -> 提供 3 选项（AI/用户/跳过） |
| 讨论陷入胶着 | **循环探测**：>2 轮讨论同一细节 -> 触发跳出引导 |
| 上下文遗忘 | **读档机制**：通过 creation-log.md 找回状态和共识 |
| 调研结果空泛 | **映射表检查**：若无法填满映射表，视为调研不合格，需补搜 |
| 文章过长 | **拆篇提示**：主动询问是否拆分，避免单篇过长导致阅读率下降 |

## 10. 使用建议

1. **不用担心被打断**：有了实时存档，即使隔天再来，AI 也能通过读取 Log 瞬间找回状态。
2. **避免纠结**：当 AI 提示"我们似乎在这个细节上讨论了很久"时，建议听从引导，先推进流程，避免完美主义陷阱。
3. **重视调研**：现在的调研不仅仅是找资料，更是在找"读者的痛点"。请认真确认 AI 提供的"读者原话"和"映射表"。
4. **主动权**：虽然 AI 会引导流程，但你可以随时叫停或跳转。
5. **模块化控制**：利用模块化写作特性，逐个模块确认，确保每个部分都符合心意。
6. **时效性确认**：调研开始时，请留意 AI 是否确认了今天的日期，这对数据准确性至关重要。
7. **多稿对比**：如果你手头有其他 AI 写的版本，不妨试试"对比分析"，取长补短。
