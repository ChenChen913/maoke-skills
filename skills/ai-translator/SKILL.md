# AI Translator - 专业英中翻译 Skill

## 概述

本 Skill 用于将英文内容翻译为高质量简体中文，支持技术文档、学术论文、博客文章等多种文档类型。核心能力包括：智能分类、术语管理、分段翻译、翻译腔检测、双格式输出。

### 适用范围

- **支持方向**：英文 → 简体中文
- **擅长领域**：AI/ML、软件开发、学术论文、技术博客、产品文档
- **输入格式**：Markdown、纯文本、PDF（通过工具读取）、URL（通过抓取）

### 不适用场景

- 文学翻译（小说、诗歌等需要文学创作能力的内容）
- 法律/合同翻译（需专业法律审核）
- 中文 → 英文翻译（本 Skill 专注英译中）
- 非英文内容（日文、法文等其他语言不在支持范围内）
- 中英混合文档：部分支持——仅翻译英文段落，中文段落保持原样

---

## 第一步：确认输入与模式选择

### 1.1 输入确认

确认用户提供的翻译内容：
- **文件路径**：读取文件内容（支持 .md / .txt / .pdf 等）
- **粘贴文本**：直接使用
- **URL**：抓取网页内容后翻译

**语言检测**：抽样检查输入内容，若检测到非英文（如日文、法文等），提示用户：`"本 Skill 仅支持英文→中文，当前内容疑似非英文，请确认。"` 若为中英混合文档，仅翻译英文段落，中文段落保持原样。

**边缘情况处理**：
- **混合文档**：若用户粘贴的文本已包含部分中文翻译，自动识别并跳过已翻译段落，仅翻译剩余英文。
- **格式损坏**：若 Markdown 格式（如表格、代码块）损坏，尝试自动修复结构后再翻译。
- **无法访问 URL**：若 URL 需要登录或无法抓取，提示用户："无法访问该链接，请直接粘贴文本内容。"

### 1.2 翻译模式

询问用户选择模式（未指定则默认**标准模式**）：

| 模式 | 适用场景 | 功能范围 |
|------|----------|----------|
| **快速** | 短文本（<500字）、非关键内容 | 直接翻译 + 基础格式化（**跳过**：术语确认、反译验证、质量报告） |
| **标准**（默认） | 技术文档、博客 | 术语管理 + 翻译腔优化 + 双格式输出 |
| **精品** | 论文、产品文档 | 标准 + 多版本验证 + 反译验证（**耗时较长**，建议用于 <3000 字的关键文档） |

**自动模式推荐**：输入 <500 字时默认推荐快速模式，用户可覆盖。

### 1.3 输出格式确认

- 默认输出：纯中文版 `translated_cn.md`
- 标准/精品模式额外输出：中英对照版 `translated_bilingual.md`
- 用户可指定仅输出其中一种

## 第二步：预翻译准备

### 2.1 智能分类

分析文档内容，自动识别双重标签：

**领域识别**（根据高频关键词）：
- AI/ML：machine learning, neural network, training, model, inference
- 软件开发：API, endpoint, deploy, container, microservice
- 学术论文：abstract, methodology, hypothesis, findings
- 商业/营销：revenue, strategy, market, ROI

**类型识别**（根据结构特征）：
- API 文档：endpoint, GET/POST, JSON 示例, request/response
- 用户手册：step, click, navigate, screenshot
- 学术论文：abstract → introduction → methodology → conclusion
- 技术博客：tutorial, how-to, example, let's
- 更新日志：version, changelog, fixed, added, deprecated

告知用户分类结果：`"已识别为 [领域] - [类型]，将使用对应翻译策略。"`

根据文档类型加载特殊翻译规则，详见 [document-types.md](document-types.md)。

### 2.2 术语准备

**步骤 1**：加载核心术语表（见 [terminology-guide.md](terminology-guide.md)）

**步骤 2**：如用户提供了自定义术语表，合并加载（用户术语优先级最高）

**步骤 3**（标准/精品模式）：
- 扫描原文提取专业术语清单
- 对不确定的术语，尝试网络搜索验证译法
- 搜索失败时，用 AI 知识推测并标记 `[待验证]`
- 新发现的术语记录到工作目录下 `discovered_terms.md`

**步骤 4**：向用户展示关键术语翻译方案，确认后开始翻译。若无高不确定性术语且用户未提供自定义术语表，可跳过确认直接翻译。

### 2.3 长文档分段

文档超过 **10000 字**时启动分段翻译（建议上限 **100000 字**，超出建议用户拆分为多个文件）：

**分段优先级**：
1. 按章节标题断开（`##` `###` 等 Markdown 标题）
2. 按段落断开（保持段落完整）
3. 保持逻辑单元完整（代码块、表格、公式不拆分）

**每段目标长度**：3000-5000 字

长文档翻译时，每完成一段向用户报告进度：`"已翻译 3/8 段..."`

**上下文传递**：
- 翻译第 N 段时，将第 N-1 段最后 2-3 句作为上下文输入
- **术语一致性**：对于术语密集型文档，同时传入当前已生成的 `discovered_terms.md` 作为额外上下文
- 提示 AI：前 2-3 句仅供理解上下文，不要翻译输出
- 所有段落译文追加写入同一目标文件

## 第三步：翻译执行

### 3.1 构建翻译提示词

根据分类结果组装提示词：

```
你是一位资深的{领域}翻译专家，精通英文和简体中文，熟悉{类型}的翻译规范。

请将以下内容从英文翻译为简体中文。

## 术语表（必须严格遵守）
{加载的术语列表}

## 翻译要求
1. 术语使用必须一致，严格按照术语表
2. 中文表达流畅自然，避免翻译腔
3. 被动语态转主动语态（"被广泛使用"→"广泛使用"）
4. 长难句（>100字）拆分为 2-3 个短句
5. 冗余表达精简（"进行训练操作"→"训练"）
6. 去除多余代词（"该模型它使用"→"该模型使用"）
7. 专业术语首次出现标注英文：机器学习（Machine Learning）
8. 中英文之间加空格：使用 PyTorch 框架
9. 使用中文标点：，。、；：""（）
10. 保留所有 Markdown 格式标记
11. 代码块内仅翻译注释，代码保持英文
12. URL、文件路径保持原样

{文档类型特殊规则}

---
{待翻译内容}
```

### 3.2 执行翻译

- **短文档**（<10000字）：一次性翻译
- **长文档**（≥10000字）：按分段计划逐段翻译，每段追加写入目标文件

### 3.3 精品模式附加：多版本验证

对每段生成 3 个版本：
- **版本 A**：侧重术语准确性
- **版本 B**：侧重语言流畅性
- **版本 C**：平衡准确与流畅

AI 自评维度（术语准确性 30% + 流畅性 30% + 格式保留 20% + 完整性 20%），选综合最优版本。

## 第四步：后处理优化

### 4.1 翻译腔检测

逐项检测（详细规则见 [translation-tone-guide.md](translation-tone-guide.md)）：

| 检测项 | 模式 | 处理方式 |
|--------|------|----------|
| 被动语态 | "被...应用/使用/认为" | 改为主动语态 |
| 多余代词 | "名词 + 它/其 + 动词" | 去除多余代词 |
| 超长句 | >100 字 | 拆分为 2-3 个短句 |
| 冗余表达 | "进行X操作""执行X过程" | 精简为 "X" |
| 标点混用 | 英文逗号/句号混入中文 | 替换为中文标点 |

### 4.2 格式规范化

- 中英文之间加空格：`使用 PyTorch 框架训练 BERT 模型`
- 中文与数字之间加空格：`训练了 100 个 epoch`
- 百分号不加空格：`准确率达到 95%`
- 使用中文标点：`，。、；：""（）`
- 代码块、表格、链接格式保持完整

### 4.3 精品模式附加：反译验证

1. 将关键段落的中文译文反译回英文
2. 对比原文与反译文，识别语义偏差
3. 修正偏差较大的段落

### 4.4 生成优化报告

在工作目录下生成 `quality_report.md`（快速模式跳过），格式如下：

```markdown
# 翻译质量报告

| 检测项 | 发现数量 | 已修正 / 待人工处理 | 说明 |
|---|---|---|---|
| 被动语态 | 5 | 5 / 0 | 已自动转为主动语态 |
| 超长句（>100字） | 2 | 2 / 0 | 已拆分为短句 |
| 待验证术语 | 3 | 0 / 3 | 详见 discovered_terms.md |
| 标点混用 | 1 | 1 / 0 | 已修正为中文标点 |
```

## 第五步：输出

### 5.1 纯中文版（translated_cn.md）

```markdown
# {文档标题}

> 翻译自：{原文来源}
> 翻译模式：{快速/标准/精品}

## 术语表

| 中文 | English | 说明 |
|------|---------|------|
| 神经网络 | Neural Network | 模拟人脑神经元的计算模型 |
| ... | ... | ... |

---

{完整中文译文}
```

### 5.2 中英对照版（translated_bilingual.md）

```markdown
# {文档标题} - 中英对照版

## 术语表
{同上}

---

## {章节标题}

{中文译文}

> **Original:** {英文原文}

---
{下一章节...}
```

### 5.3 术语表生成

- 统计文档中专业术语出现频次
- 筛选出现 ≥3 次的术语，按频次排序取前 20-30 个
- 每个术语包含：中文译法、英文原词、简要说明

## 第六步：反馈与迭代

### 6.1 用户反馈识别

| 用户说 | 意图识别 | 处理方式 |
|--------|----------|----------|
| "这段太生硬" | 翻译腔 | 增加流畅性权重重翻该段 |
| "术语不对" | 术语错误 | 重新调研术语，修正译文 |
| "太长了" | 需精简 | 去除修饰词，精简表达 |
| "看不懂" | 可读性差 | 重组语言，增加说明 |
| "和上文重复" | 分段重复 | 检查并去除重复内容 |

### 6.2 定向优化流程

1. 定位问题段落
2. 分析问题类型
3. 生成 2-3 个优化版本
4. 展示给用户选择
5. 用户确认的术语修正更新到 `discovered_terms.md`

---

## 第七步（可选）：外部提示词分析

当用户提供外部翻译提示词进行对比时：

1. 接收并记录外部提示词内容
2. 与本 Skill 的翻译流程进行 4 维度对比分析：
   - 翻译流程差异
   - 创新点识别
   - 优劣势对比
   - 可集成性评估
3. 生成分析报告
4. 提供集成建议供用户决策
5. 用户确认集成后，更新对应模块

API, SDK, HTTP, JSON, REST, GraphQL, URL, HTML, CSS, Python, Docker, Git, GitHub, CPU, GPU, RAM, SQL, CI/CD, OAuth, JWT, WebSocket
