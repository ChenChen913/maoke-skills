# 翻译腔检测与优化指南

本文档包含翻译腔的检测模式和修复规则，在翻译后处理阶段使用。

---

## 什么是翻译腔

翻译腔是指译文虽然语法正确，但不符合中文母语者的表达习惯，读起来像"翻译过来的文字"而非自然书写的中文。

---

## 检测项 1：被动语态

### 常见模式

| 翻译腔（避免） | 自然表达（推荐） |
|----------------|------------------|
| 被广泛应用于... | 广泛应用于... |
| 被认为是... | 被公认为... 或 普遍认为... |
| 建议使用该方法 | 建议你使用该方法 / 请使用该方法 |
| 它被设计为... | 其设计目标是... |

### 英文被动语态转换规则

| 英文 | 翻译腔 | 自然中文 |
|------|--------|----------|
| It is recommended that... | 被推荐... | 建议... / 请... |
| It should be noted that... | 应该被注意... | 需要注意... / 注意... |
| It can be seen that... | 能够被看出... | 可以看出... / 由此可见... |
| X is used to... | X 被用来... | X 用于... |
| X was developed by... | X 被...开发 | ...开发了 X |

### 保留被动语态的情况

以下情况被动语态是自然的，不需要修改：
- "被证明是有效的"
- "被广泛认可"
- "被正式发布"

## 检测项 2：多余代词

### 常见模式

| 翻译腔（避免） | 自然表达（推荐） |
|----------------|------------------|
| 该模型它使用了... | 该模型使用了... |
| 这个算法其复杂度为... | 这个算法的复杂度为... |
| 该框架它提供了... | 该框架提供了... |
| 这种方法其优势在于... | 这种方法的优势在于... |

### 规则

中文中，主语明确时不需要重复使用代词。检测"名词 + 它/其 + 动词"模式，去除中间的代词。

---

## 检测项 3：超长句

### 判断标准

| 字数 | 级别 | 处理方式 |
|------|------|----------|
| ≤80 字 | 正常 | 无需处理 |
| 80-100 字 | 注意 | 如有自然断点可拆分 |
| 100-150 字 | 警告 | 建议拆分 |
| >150 字 | 严重 | 必须拆分 |

### 拆分方法

1. 找到从句边界（which, that, where, when 等翻译后的位置）
2. 在逻辑转折处断开
3. 每句保持一个核心意思
4. 使用连接词保持逻辑：因此、同时、此外

### 示例

**拆分前（翻译腔）**：
> 该模型在包含从互联网收集的超过 1000 万个文档的大规模数据集上进行训练，使用了最新的 Transformer 架构，并在多个基准测试中取得了最先进的性能。

**拆分后（自然中文）**：
> 该模型在大规模数据集上训练，数据集包含从互联网收集的超过 1000 万个文档。模型使用了最新的 Transformer 架构，并在多个基准测试中取得了业界领先的性能。

## 检测项 4：冗余表达

### 常见模式

| 冗余表达（避免） | 精简表达（推荐） |
|------------------|------------------|
| 进行训练操作 | 训练 |
| 执行优化过程 | 优化 |
| 实施测试工作 | 测试 |
| 进行数据处理 | 处理数据 |
| 对...进行分析 | 分析... |
| 对...做出评估 | 评估... |
| 实现了...的功能 | 支持... / 能够... |
| 具有...的能力 | 能够... / 支持... |

### 规则

检测"进行/执行/实施 + X + 操作/过程/工作"模式，精简为动词本身。

---

## 检测项 5：标点符号

### 中文标点规范

| 正确（中文标点） | 错误（英文标点） |
|------------------|------------------|
| ， | , |
| 。 | . |
| ；| ; |
| ：| : |
| ""| "" |
| （）| () |
| 、 | 用于并列 |

### 中英文混排空格规范

| 场景 | 正确 | 错误 |
|------|------|------|
| 中文+英文 | 使用 PyTorch 框架 | 使用PyTorch框架 |
| 中文+数字 | 训练了 100 个 epoch | 训练了100个epoch |
| 数字+百分号 | 准确率 95% | 准确率 95 % |
| 数字+单位 | 大小为 128 MB | 大小为128MB |
| 英文标点 | 保持英文标点 | 不替换 |

### 专业术语首次标注

| 场景 | 格式 |
|------|------|
| 首次出现已翻译术语 | 机器学习（Machine Learning） |
| 后续出现 | 机器学习 |
| 首次出现保持英文术语 | Transformer（一种注意力机制架构） |
| 后续出现 | Transformer |
| 缩写术语 | API（Application Programming Interface，应用程序接口） |
| 缩写后续 | API |

## 优化执行流程

翻译完成后按以下顺序执行优化：

**第 1 轮：标点符号修正**
- 替换英文标点为中文标点
- 处理中英文混排空格
- 耗时短，优先处理

**第 2 轮：被动语态优化**
- 识别不自然的被动语态
- 改为主动语态
- 保留自然的被动语态

**第 3 轮：长句拆分**
- 识别超长句
- 在逻辑断点处拆分
- 保持语义完整

**第 4 轮：冗余表达精简**
- 识别冗余模式
- 精简为核心动词
- 提升简洁性

**第 5 轮：整体流畅性检查**
- 去除多余代词
- 调整语序使其符合中文习惯
- 确保上下文连贯
