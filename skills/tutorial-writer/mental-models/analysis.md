# 思维模型：拆解与分析类

---

## 第一性原理（First Principles Thinking）

**适合场景**：读者对某个概念有根深蒂固的错误认知，或该概念容易被类比误导时。

**基本思路**：
剥离所有类比、假设和经验总结，回到最基本的、不可再分的事实，然后从这些事实出发重新构建理解。

**写作步骤**：
1. 指出"常见但不准确的理解"是什么
2. 拆解出该概念最基本的事实或定义（不依赖任何类比）
3. 从这些基本事实出发，一步步推导出正确的理解
4. 最后验证：用推导出的理解解释一个具体的现象或场景

**示例结构**：
> "很多人认为 X 就是 Y，但这个类比在以下情况会失效……真正的底层逻辑是……"

## MECE 原则（Mutually Exclusive, Collectively Exhaustive）

**适合场景**：需要对一个主题做全面分类讲解，且各子类之间不应有重叠时。

**基本思路**：
将一个大主题拆解为若干子类，要求"互不重叠"（每个知识点只属于一个类别）、"合计完整"（所有子类合在一起覆盖整个主题，没有遗漏）。

**写作步骤**：
1. 确定讲解主题的边界（什么在范围内，什么不在）
2. 设计分类维度（按功能、按使用场景、按复杂度等）
3. 检查：每个子类是否有重叠？所有子类加在一起是否完整覆盖？
4. 按子类顺序逐一讲解

**适用提示**：当一个概念有多种变体、多个子命令、多种用法时，MECE 能帮助读者建立清晰的分类心智模型。

## 抽象阶梯（Ladder of Abstraction）

**适合场景**：概念层次复杂，读者容易在"太抽象"和"太具体"之间迷失时。

**基本思路**：
Jerome Bruner 提出的概念——任何知识都可以在从具体到抽象的"阶梯"上定位。教程可以在阶梯上"上下移动"：先在具体层给出可感知的实例，再上升到抽象层提炼原则；或者先给出抽象原则，再下降到具体层用实例验证。

**写作步骤（从下往上）**：
1. 从一个非常具体的、读者能直接操作或观察的例子开始
2. 描述这个例子展示了什么现象
3. 提炼出该现象背后的规律（中层抽象）
4. 进一步提炼出可推广的原则（高层抽象）
5. 用另一个具体例子验证这个原则

**写作步骤（从上往下）**：
1. 先陈述抽象原则
2. 给出一个具体的中层例子
3. 再下降到可直接运行的代码或操作

## 组块化（Chunking）

**适合场景**：内容信息密度大，如果线性排列会造成读者认知过载时。

**基本思路**：
将大量信息组织成有意义的"组块"，每个组块内部紧密相关，组块之间有清晰的边界。人类短期记忆更擅长处理少量组块而非大量零散信息。

**写作步骤**：
1. 梳理所有需要传达的知识点
2. 按照"相关性"将知识点分组，每组 3–7 个点为宜
3. 为每个组命名（即章节标题），标题要能概括该组的核心
4. 在组与组之间写过渡句，说明"前一组讲了什么，下一组要讲什么"
5. 每一组内部的讲解顺序同样遵循认知规律（从简单到复杂）

**适用提示**：组块化适合作为整篇教程的结构设计原则，而不只是某一章节的写法。
