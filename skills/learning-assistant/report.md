# 学习助手 Skill - 技术报告

## 1. Skill 概述

**学习助手 (Learning Assistant)** 是一个全面、模块化的技能，旨在协助用户掌握技术主题（CS, AI, ML, SE）。它将 AI 转变为一个结构化的导师，管理学习进度、调整教学策略，并在不同会话之间保持长期上下文。

### 关键特性
- **结构化学习**：自动生成学习计划、抽认卡和知识总结。
- **上下文感知**：追踪用户画像 (`USER_PROFILE.md`) 和学习历史 (`LEARNING_INDEX.md`)。
- **适应性教学**：在“苏格拉底”、“面试”和“费曼”模式之间切换。
- **会话连续性**：自动将进度保存到 `ANCHOR` 文件以恢复长任务。
- **严格审计**：自我修正输出的准确性和完整性。

---

## 2. 使用指南

### 如何触发
当检测到学习相关的意图时，技能会自动激活。您也可以显式触发：
- **开始学习**：“帮我学 React”、“解释 Transformer 架构”。
- **复习**：“复习我最近的主题”、“考考我 Python”。
- **计划**：“为 Go 语言创建一个 4 周的学习计划”。
- **分析**：上传代码文件或 PDF 并说“分析这个”。
- **面试准备**：“模拟高级 Java 职位的面试”。

### 互动模式
1. **标准模式**（默认）：结构化解释，包含定义、用例和示例。
2. **苏格拉底模式**：“帮我深度理解” -> AI 提出引导性问题而不是直接给出答案。
3. **面试模式**：“面试我” -> AI 扮演面试官并点评回答。
4. **费曼模式**：“测试我的理解” -> 用户向 AI 解释，AI 点评。

### 文件系统
该技能维护一个结构化的工作区：
- `USER_PROFILE.md`：追踪您的级别、目标和已掌握的技能。
- `LEARNING_INDEX.md`：所有学习主题和复习时间表的全局索引。
- `YYYYMMDD_ANCHOR_[主题].md`：用于恢复会话的检查点。
- `YYYYMMDD_[类型]_[主题].md`：生成的学习产物（如 `20240315_PLAN_Python.md`）。

## 3. 设计原则

### 模块化架构
该技能不是一个单一的巨大提示词，而是拆分为：
- **SKILL.md**：“内核”（导航、全局规则）。
- **references/**：“库”（按需加载）。
  - `01-knowledge-analysis.md`：输出格式。
  - `02-interaction-modes.md`：教学逻辑。
  - `03-file-templates.md`：标准化文件结构。
  - ...更多。

**好处**：减少 Token 使用（上下文效率）并防止逻辑幻觉。模型仅在您实际粘贴代码时才读取 `04-code-adaptation.md`。

### 上下文管理（“锚点”系统）
LLM 的记忆有限。本技能通过以下方式解决：
1. **快照**：定期更新 `USER_PROFILE.md`。
2. **锚点**：在会话结束时生成 `ANCHOR` 文件，总结状态。
3. **索引**：`LEARNING_INDEX.md` 作为查找过去文件的地图。

### 合规与安全
- **严格输出逻辑**：使用 IF-THEN 伪代码强制确定性行为。
- **审计就绪**：构建时通过了严格的质量审计（触发词覆盖、文件结构、双语支持）。

## 4. 详细模块细分

### 核心逻辑 (`SKILL.md`)
- **会话清单**：强制执行严格的初始化（检查日期 -> 读取画像 -> 检查复习）。
- **漂移检测**：如果对话偏离目标太远，会发出警告。

### 知识解析 (`01-knowledge-analysis.md`)
- 定义了 12 个分析维度（定义、用例、陷阱等）。
- 根据用户请求调整深度（快速 vs. 深度）。

### 互动模式 (`02-interaction-modes.md`)
- 实现主动学习策略。
- **苏格拉底**：促进批判性思维。
- **面试**：模拟压力。
- **费曼**：强制表达。

### 模板 (`03-file-templates.md`)
- 所有文件格式的单一事实来源。
- 确保生成的文档的一致性。

### 领域适配 (`05-domain-adaptation.md`)
- 根据主题定制解释：
  - **CS**：注重可视化和复杂度。
  - **AI**：注重数学 + 直觉。
  - **SE**：注重权衡和场景。

### 调研策略 (`06-research-strategy.md`)
- 分层方法：标准（文档/博客）vs. 深度（论文/源代码）。
- 如果搜索失败，优雅降级。
